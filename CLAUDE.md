## Claude Workspace Guide

This file mirrors the workspace system defined in `CODEX.md` so another AI assistant can work consistently in the same environment.

## Startup Routine

On every new session:

1. Read `CLAUDE.md`.
2. Read `CODEX.md`.
3. Read `MEMORY.md`.
4. Scan the workspace structure.
5. Identify important files, folders, docs, prompts, projects, and skills before starting work.

## Workspace Purpose

This workspace is a personal AI system for Mostafa Jarjour.
It is designed to grow over time and support:

- project work,
- reusable skills,
- prompt systems,
- durable documentation,
- templates,
- context files,
- persistent memory.

## Core Files

- `CODEX.md` - main agent operating guide for the workspace.
- `MEMORY.md` - persistent memory and stable preferences.
- `CLAUDE.md` - compatibility guide for Claude or similar assistants.

## Root Folder Structure

- `context/` - background information and supporting context.
- `docs/` - durable documentation and reference material.
- `projects/` - active and future project folders.
- `prompts/` - reusable prompt files and templates.
- `skills/` - reusable skill instructions and learned workflows.
- `templates/` - reusable starter templates.

## Working Rules

- Treat the workspace as a living system.
- Do not assume the structure is static.
- When new important files or folders appear, understand them before acting.
- Keep the workspace organized and reusable.
- Prefer updating core instructions when stable patterns emerge.

## Skills Rule

When a new repeatable capability is learned:

1. decide whether it belongs in `skills/`,
2. update `MEMORY.md` if it reflects a lasting preference or workflow,
3. update `CODEX.md` if it changes the workspace operating model.

## Project Rule

Before working inside a project:

1. inspect that project's files,
2. read its docs and task notes if they exist,
3. understand its current status,
4. avoid making assumptions based only on root-level files.

Current known active project:

- `projects/misso-clinic/` - real project folder currently present in the workspace.

## Consistency Rule

If `CLAUDE.md`, `CODEX.md`, and `MEMORY.md` drift apart, the newest stable workspace structure and user preferences should be reflected across them.
